# Persona System

사이트마다 고유 페르소나를 부여하여 같은 상품이라도 다른 관점의 콘텐츠를 생성하는 시스템.

## 페르소나 구조

```typescript
persona: {
  name: string;        // 페르소나 이름
  age: number;         // 나이
  concern: string;     // 주요 관심사
  expertise: string;   // "초중급" | "중급" | "전문가"
  tone: string;        // 글쓰기 톤
  bio: string;         // 배경 스토리
}
```

**설정 파일**: `configs/sites-config.json`

## 페르소나가 콘텐츠에 미치는 영향

### 프롬프트 영향
1. **전문성 수준**: 초중급은 쉬운 용어, 전문가는 성분 분석 깊이
2. **관심사 초점**: 각 페르소나의 concern에 맞는 내용 강조
3. **톤 일관성**: 담백/전문/친근/유머/감성에 따른 문체
4. **배경 활용**: bio 정보로 신뢰성 구축

### 예시: 같은 비타민C 제품
- **하준** (담백): "야근 많은 직장인이라 피로 회복 위주로 봤습니다"
- **서연** (전문): "아스코르빈산 함량과 흡수율을 비교 분석해보면..."
- **지은** (친근): "아이 수유 중인데 이거 먹어도 될까 고민했어요"
- **민수** (유머): "운동 후 먹는 거니까 흡수 속도가 관건이지 않겠습니까"
- **영숙** (감성): "나이 들수록 건강이 재산이라는 걸 느끼면서..."

## 콘텐츠 다양성 전략

### 조합 계산
- 5개 페르소나 × 8개 앵글 = **40가지 고유 콘텐츠 변형** (상품 기준)
- 리뷰 윈도우 분산으로 같은 앵글이어도 참조 리뷰가 다름

### E-E-A-T 구축
| EEAT 요소 | 구현 방식 |
|-----------|-----------|
| **Experience** | 리뷰 데이터를 1차 소스로 활용 |
| **Expertise** | 페르소나 전문성 수준에 맞는 깊이 |
| **Authoritativeness** | 자체 도메인 + 일관된 페르소나 |
| **Trustworthiness** | 장단점 균형, 증거 기반 표현 |

## 페르소나 확장 가이드

새 페르소나 추가 시 `configs/sites-config.json`에 사이트 설정과 함께 추가:

```json
{
  "site_slug": "new-site",
  "domain": "new-site.site",
  "site_title": "사이트명",
  "persona": {
    "name": "캐릭터명",
    "age": 35,
    "concern": "주요 관심사",
    "expertise": "중급",
    "tone": "담백",
    "bio": "배경 스토리 (2-3문장)"
  }
}
```

### 효과적인 페르소나 설계 원칙
1. **연령대 분산**: 20대~50대로 타겟 독자층 커버
2. **관심사 차별화**: 겹치지 않는 고민/니즈 설정
3. **전문성 스펙트럼**: 초보~전문가로 다양한 깊이
4. **톤 다양성**: 독자가 선호하는 문체별로 분산
5. **구체적 bio**: 신뢰할 수 있는 구체적 배경 스토리
