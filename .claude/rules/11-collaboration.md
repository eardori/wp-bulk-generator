# Collaboration Rules

## 1. 멤버 간 요청 보드 (`docs/tasks/`)

### 요청 생성 트리거
| 감지 상황 | Claude의 동작 |
|-----------|---------------|
| "Kevin에게 {작업} 요청해줘" | kevin-inbox.md에 요청 추가 |
| "Justin이 {작업} 해야 해" | justin-inbox.md에 요청 추가 |
| "Hoon 승인 요청: {내용}" | hoon-inbox.md에 요청 추가 |
| 자기 권한 밖의 파일 수정 필요 | 해당 소유자 inbox에 요청 생성 제안 |

### 요청 작성 형식
```markdown
| T-{NNN} | {YYYY-MM-DD} | {요청자} | {우선순위} | {내용} | {관련파일} | {비고} |
```

### 요청 상태 전환
1. 생성 → **대기 중** 테이블에 추가
2. 작업 시작 → **진행 중** 테이블로 이동
3. 작업 완료 → **완료** 테이블로 이동
4. 전체 인덱스(`docs/tasks/index.md`)도 동시 업데이트

## 2. 기능 진행 대시보드 (`docs/status.md`)

### 자동 업데이트 제안
| 상황 | Claude의 동작 |
|------|---------------|
| API route 구현 완료 | "status.md도 업데이트할까요?" 제안 |
| 새 스크래퍼 추가 | 스크래핑 소스 목록 업데이트 제안 |
| 새 기능 개발 시작 | 기능 행 추가 제안 |
| 버그 수정 완료 | 해당 기능 상태 업데이트 제안 |

## 3. 세션 종료 요약

### 트리거
"마무리", "끝", "오늘 여기까지" 등 종료 의사 표현 시:

```
━━━ 세션 요약 ━━━━━━━━━━━━━━━━━━━━━━━

📅 {날짜} — {멤버 이름} ({역할})

✅ 완료한 작업:
• {작업} — {관련 파일}

📬 다른 멤버에게 전달할 사항:
• → {멤버}: {내용}
(없으면: "전달 사항 없음")

⏭️ 다음에 이어서 할 작업:
• {다음 작업}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 4. 의사결정 기록 (ADR)

### 트리거
| 감지 상황 | Claude의 제안 |
|-----------|---------------|
| 기술적 결정 발생 | "이 결정을 ADR로 기록할까요?" |
| 라이브러리 선택 논의 | "선택 근거를 ADR로 남길까요?" |
| "왜 이렇게 했어?" 질문 | 관련 ADR 조회 또는 신규 작성 제안 |

### ADR 파일 형식
`docs/decisions/ADR-{NNN}-{title}.md`
